1. tuliskan tentang perbedaan mendasar antara compile-time error dan runtime error!​


Compile-time Error:

Biasanya ini di bahasa pemograman yang memiliki sistem compiler, dan bukan interpreter

Dari katanya "compile" yang memiliki arti menyusun, bisa terjadi karena ada kesalah dalam script yang membuat penyusunan gagal, contohnya

C code, kesalahan: nputchar is not defined

#include <stdio.h>

#include <stdlib.h>

int main() {

nputchar("Hello, World!");

}

Run-time Error:

Ini berada di semua bahasa pemograman karena kesalahan yang belum di deteksi oleh interpreter atau compiler dikarenakan pernyataan fungsi atau pernyataan jika, contohnya

#Python code

var = 0

answer = eval(input())

if answer == True:

var = 1/0

31. Job D datang pada jam 08.20 dengan runtime sebesar 30 menit, 10 menit sebelum Job D telah datang job E dengan runtime 20 menit lebih besar daripada runtime job D. Job A datang pada jam 08.40 dengan runtime sama besar dengan runtime Job D. Ternyata 5 menit sebelum job E datang telah datang pula Job C dengan runtime 20 menit, sedangkan job B datang lebih awal 40 menit dari job A dengan runtimenya 40 menit.Tentukan Average Turn Arround Time (AVR-TAT) dari job-job di atas dengan metode: FCFS, SJF, PSJF, HPRN dan RR (Q=10 Menit). buatkan tabel dan grafik gambar.​


Jawaban:

Untuk menghitung Average Turnaround Time (AVR-TAT) dari job-job menggunakan metode FCFS, SJF, PSJF, HPRN, dan RR, kita dapat membuat tabel dan menghitung waktu proses selesai (completion time) untuk setiap job. Berikut adalah tabel yang menunjukkan waktu kedatangan (arrival time), runtime, waktu mulai (start time), waktu selesai (completion time), dan turnaround time (TAT) dari setiap job:

```

| Job | Arrival Time | Runtime | Start Time | Completion Time | Turnaround Time |

|-----|--------------|---------|------------|-----------------|-----------------|

| A | 08:40 | 30 | - | - | - |

| B | 08:00 | 40 | - | - | - |

| C | 08:15 | 20 | - | - | - |

| D | 08:20 | 30 | - | - | - |

| E | - | - | - | - | - |

```

Berikut adalah langkah-langkah untuk mengisi tabel ini dengan metode FCFS, SJF, PSJF, HPRN, dan RR:

1. FCFS (First-Come, First-Served):

- Urutkan job berdasarkan kedatangan (arrival time).

- Mulai dari job pertama, tetapkan waktu mulai (start time) job tersebut sesuai dengan waktu kedatangan.

- Hitung completion time dengan menambahkan runtime job dengan waktu mulai.

- Hitung turnaround time dengan mengurangi arrival time dari completion time.

2. SJF (Shortest Job First):

- Urutkan job berdasarkan runtime (dari yang terkecil hingga terbesar).

- Mulai dari job pertama, tetapkan waktu mulai (start time) job tersebut sesuai dengan waktu kedatangan.

- Hitung completion time dengan menambahkan runtime job dengan waktu mulai.

- Hitung turnaround time dengan mengurangi arrival time dari completion time.

3. PSJF (Preemptive Shortest Job First):

- Urutkan job berdasarkan runtime (dari yang terkecil hingga terbesar).

- Mulai dari job pertama, tetapkan waktu mulai (start time) job tersebut sesuai dengan waktu kedatangan.

- Jika ada job lain dengan runtime lebih kecil yang datang sebelum job tersebut selesai, lakukan preemption dan tetapkan waktu mulai yang baru.

- Hitung completion time dengan menambahkan runtime job dengan waktu mulai.

- Hitung turnaround time dengan mengurangi arrival time dari completion time.

4. HPRN (Highest Priority Ratio Next):

- Hitung priority ratio untuk setiap job, yaitu runtime job dibagi dengan selisih antara waktu sekarang dengan waktu kedatangan job tersebut.

- Urutkan job berdasarkan priority ratio (dari yang tertinggi hingga terendah).

- Mulai dari job pertama, tetapkan waktu mulai (start time) job tersebut sesuai dengan waktu kedatangan.

- Hitung completion time dengan menambahkan runtime job dengan waktu mulai.

- Hitung turnaround time dengan mengurangi arrival time dari completion time.

5. RR (Round Robin):

- Tetapkan waktu quantum (Q).

- Urutkan job berdasarkan kedatangan

(arrival time).

- Mulai dari job pertama, tetapkan waktu mulai (start time) job tersebut sesuai dengan waktu kedatangan.

- Jika runtime job melebihi waktu quantum, hitung completion time dengan menambahkan waktu quantum dengan waktu mulai.

- Jika runtime job tidak melebihi waktu quantum, hitung completion time dengan menambahkan runtime job dengan waktu mulai.

- Jika job belum selesai, pindahkan job tersebut ke akhir antrian dan lanjutkan ke job berikutnya.

- Hitung turnaround time dengan mengurangi arrival time dari completion time.

Setelah tabel diisi, kita dapat menghitung Average Turnaround Time (AVR-TAT) dengan menjumlahkan semua turnaround time dan membaginya dengan jumlah job.

Jawaban:

34. Apa yang terjadi jika konstruktor memiliki return type? a. Terjadi compilation error b. Runtime error c. Kompilasi dan eksekusi bisa berjalan dengan lancar d. Hanya return type String yang diperbolehkan e. Hanya kompilasi yang berhasil


Konstruktor class didefinisikan sebagai suatu metode yang begruna untuk menganalisis variabel-variabel instans yang akan dimiliki oleh pbjek. Ketika konstruktor memiliki return type maka hanya return type string yang diperbolehkan.  (D)

Pembahasan

Konstruktor class didefinisikan sebagai suatu metode yang begruna untuk menganalisis variabel-variabel instans yang akan dimiliki oleh objek. konstruktor ini digunakan pada saat proses instansiasi kelas menjadi objek. Peran dari metode ini yakni akan memberikan suatu nilai awal ketika objek dibentuk. Ketika program dijalankan konstruktor akan memberikan hasil nilai awal pada saat perintah new atau membuat suatu objek. Konsturltor pada umumnnya digunakan untuk menyiapkan data class. Dalam hal tersebut perlu membuat suatu para meter sebagai inputan untuk konstruktor. Ketika konstruktor memiliki return type maka hanya return type string yang diperbolehkan.
